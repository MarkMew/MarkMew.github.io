B2C サービスや外部向け Web サイトの運営に関わっている人なら、

SEO のさまざまな調整方法にはかなり詳しいはずです。

AI 時代が到来したことで、

ユーザーは検索テクニックを磨くだけではなくなりました。

今では、さまざまなプロンプトをどう調整するかを学び始めています。

AI 時代の到来によって、

SEO もまた新しい変化を迎えようとしています。

これまでの SEO は、Google の検索結果 1 ページ目に表示されることを目指していました。

しかし今後は、AI ツールにコンテンツを収録され、引用されるにはどうすればよいかが重要になります。

AI に収録されるコンテンツの価値はさらに高まり、SEO 4.0 の時代が始まっています。

## SEO

### SEO とは

SEO は Search Engine Optimization の略で、日本語では`検索エンジン最適化`と呼ばれます。

簡単に言えば、検索エンジンに自分の Web サイトを正しく認識してもらうための取り組みです。

中国では百度站長工具が使われます。

中国以外の地域では、Google Search Console（旧 Google Webmaster Tools）を使って Web サイトのインデックス状況を確認するのが一般的です。

検索エンジンに収録されるには、通常 2 つのプロセスを経る必要があります。

1 つ目はクロール（Crawling）です。

Google 以外にも、

すべての検索エンジンにはそれぞれ独自のクローラーがあります。

クローラーは Web サイト上のリンクをたどり、

関連するコンテンツを取得します。

2 つ目はインデックス（Indexing）です。

クローラーが取得したページ内容を検索エンジンが分析・分類し、

最終的に保存してインデックスへ登録します。

SEO はマーケティング部門の仕事だと考える人もいます。

この考え方は、半分は正しいと言えます。

どのようにコンテンツを宣伝し、広げるかを考えるのは、確かにマーケティング部門の仕事です。

ただし多くの場合、SEO の成果は Web サイトそのものの体質に大きく左右されます。

ページの応答が遅い、使いにくい、コンテンツが乏しい。

こうした状態は、すでに SEO に大きな影響を与えています。

### SEO の変遷

> 以下では SEO 1.0、SEO 2.0、... という表現を使いますが、
> 実際には Web 1.0 や Web 2.0 のように標準化された区分ではありません。
> 一般的にこのように呼ばれるわけでもなく、
> 年代で明確に区切られているわけでもありません。
> よく使われるのは、キーワード時代、PageRank 時代、セマンティック検索のように、
> 実際の SEO の手法や技術的特徴に基づいた呼び方です。

#### SEO 1.0 キーワードの時代（約 1995-2005）

この時代は、Web のあらゆるものがまだ始まったばかりでした。

Web サイトもコンテンツも、今から見るとかなり素朴でした。

キーワード密度が高ければ高いほど、

検索順位も上がりやすい時代です。

子どものころに Web ページを見ていると、

本文とは関係のない文字が大量に並んでいるページを見かけることがありました。

あるいは、うっかりソースコードを開くと、

画面には表示されていない大量の文字がびっしり書かれていることもありました。

これらはまさにこの時代の産物です。

当時の検索エンジンは、コンテンツを理解する能力がまだ限られていました。

そのため Web サイトは、大量の流行語、専門用語、意味の薄い言葉、隠しテキストなどを使い、

検索に引っかかる可能性を高めようとしていました。

この時期、Google の共同創業者である Larry Page と Sergey Brin が PageRank を発表しました。

PageRank は、Web ページ同士のリンク関係を分析し、ページの重要度を判断する仕組みです。

順位の高い Web サイトが別のサイトへリンクすると、

リンクされた側のサイトも評価が上がる可能性があります。

そこで、互いにリンクを貼り合う相互リンクによって、

検索順位を上げようとする動きが広がりました。

#### SEO 2.0 リンク権威の時代（約 2005-2015）

PageRank の影響を受けて、

Web サイト同士が互いにリンクし、参照し合うようになりました。

`友好リンク` や `おすすめサイト` のような欄もよく見られるようになります。

ただし、単純に他人のサイトへリンクするだけでは、自分の順位向上にはあまりつながりません。

当時はポータルサイトも成長し始めていたため、

自分の Web サイトをポータルサイトへ登録申請するという戦略もありました。

ポータルサイトのディレクトリに URL が掲載されれば、

検索順位をさらに引き上げられる可能性がありました。

ここまでの話は、まだ比較的正攻法に近いものです。

しかし次第に、やり方が歪んでいきます。

たとえばブログネットワークです。

リンクがそれほど重要なら、

自分でサイトを作って自分のサイト同士をリンクすればよいのではないか。

そう考えた人たちによって、大量の複製ブログや複製サイトが作られました。

その後、コンテンツファームも登場します。

あなたのサイトに良いコンテンツがあるなら、

その内容をコピーしてしまえばよい。

そうすれば、検索エンジンはどちらが本来のサイトなのか判断しにくくなります。

流入や順位向上を得たあと、

サイトを売却したり、大量の広告を差し込んで収益化したりするわけです。

低品質なコンテンツに対抗するため、

Google もアルゴリズムの調整にかなりの労力を費やしました。

#### ソーシャルシグナル

一時期、Web サイトはソーシャル上での話題性を非常に重視していました。

この時期は、おおよそ SEO 2.0 から SEO 3.0 にまたがっています。

ただし他の時期のように明確な代表的成果物があるわけではないため、

ここでは SEO 2.5 とはせず、中間的な位置づけにしています。

SEO 2.0 の核心的な問題は、外部リンクが買われたり操作されたりすることでした。

そこで Google は、新しい「信頼シグナル」を探し始めます。

Facebook が世界的に普及したあと、

Plurk、Twitter、MySpace、tumblr など、

多くのソーシャルサービスが次々に登場しました。

SEO 業界では、こう考えられるようになります。

多くの実在するユーザーがある記事を共有するなら、それは良質なコンテンツである証拠ではないか。

そのためこの時期には、会社が Facebook に投稿したあと、

社員に一斉投稿や記事の共有を強制するような、少し滑稽な現象もありました。

当時、人々は何を観察していたのでしょうか。

上位表示される記事には、次のような特徴がありました：
- Facebook で大量にシェアされている
- Twitter で大量にリポストされている
- ソーシャル上で多く議論されている

ただし、相関関係は因果関係ではありません。

実際には、良いコンテンツだから多くの人に共有されたのでしょう。

共有されたあと、さらに引用され、議論される。

その結果として順位が上がったと考えるほうが自然です。

みんなが無理やり共有したから Google が順位を上げた、

という話ではありません。

#### SEO 3.0 コンテンツ品質と検索意図の時代（約 2015-2022）

2014 年には大きな節目がありました。

HTML 5 がついに登場したことです。

それ以前は、

見た目の派手な Web サイトにはまだ Flash が多く使われていました。

HTML で使える要素も限られており、

多くのサイトはほとんど `<div>` だけで構成されていました。

しかし HTML 5 の登場によって、

多くのセマンティックタグが利用できるようになります。

たとえば `<section>`、`<article>`、`<header>`、`<footer>`、`<nav>`、`<aside>`、`<figure>`、`<figcaption>` などです。

これらのセマンティックタグによって、

クローラーや検索エンジンは Web サイト全体の構造をより理解しやすくなりました。

Microdata の登場も重要でした。

`<p>` などの要素に itemscope や itemprop を追加することで、

検索エンジンは Web サイトのデータをより構造化して扱えるようになり、

サイト全体の内容を理解しやすくなりました。

#### SEO 4.0

そして現在、私たちは AI の時代に入っています。

SEO 1.0 時代の明らかに淘汰されるべき手法を除けば、

これまで述べてきた多くの要素を、あらためて重視する必要が出てきました。

最近、ワールドカップが始まりました。

Plurk では以前、「ドイツが優勝したら裸で走る」というネタが流行したことがあります。

また、馮翊綱と宋少卿の相声に出てくる「葱をたっぷり散らす」というフレーズが、

Plurk 上でさまざまな JavaScript エフェクトとして具体化されたこともありました。

かつては比較的重要ではないと思われていたソーシャルシグナルも、

再び注目すべきものになるかもしれません。

### AI 時代に SEO 戦略をどう調整するか

これまで SEO について学ぶとき、

私は Google の `検索エンジン最適化（SEO）スターター ガイド` を読むよう教えられてきました。

今でも、やるべきことの大部分は変わっていないと思います。

基本は、やはり Web サイトのコンテンツを充実させることです。

そのうえで、HTML 5 のセマンティックタグを適切に使ってページを書き、

必要に応じて Microdata を組み合わせ、ページ全体の構造を強化します。

次に、Threads などで記事を公開し、

意図的かどうかにかかわらず議論を生み出すことも考えられます。

もちろん、後者だけに注力するという選択もあります。

しかし、話題はいずれ流れていきます。

私は、多くの基礎は時代が変わっても変わらないと考えています。

良質なコンテンツを提供し、サイト全体の構造を強化し、ユーザーが使いやすい状態にする。

そうして初めて、検索順位は自然に上がり、

AI にも自然に収録されるようになるはずです。

## まとめ

SEO の本質は、特定のアルゴリズムに迎合することではありません。

コンテンツを正しく理解してもらい、信頼できる形で引用され、ユーザーの課題を本当に解決することです。

キーワードの時代、リンク権威の時代、コンテンツ品質と検索意図の時代。

それぞれの変化が私たちに教えていることは同じです。

短期的なテクニックは一時的に効果を持つかもしれません。

しかし長期的には、コンテンツの品質、構造の明確さ、信頼性こそが本質です。

AI Agent の登場は、SEO が消えることを意味しません。

むしろ SEO の目標が、「検索結果に表示されること」から「AI に理解され、引用され、推薦されること」へ広がったということです。

だからこそ、次にやるべきことはより明確です。

価値あるコンテンツを書き続けること。

明確なサイト構造とセマンティックタグで、機械が理解しやすい形にすること。

そして、実際のソーシャル上の議論を通じてコンテンツの信頼性を積み上げること。

人にとって役立つ Web サイトであってこそ、

検索エンジンや AI にとっても、収録する価値のあるサイトになるのだと思います。
